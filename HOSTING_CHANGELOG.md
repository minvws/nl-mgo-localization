## [v0.15.0]

### Added

- CONF: `app.on_startup_cron_commands` :list[str] (e.g. `search-index:update`)
    - Configures a search index update cron task to run on startup of the application.
    - Default: `[]`.

- CONF: `search_indexation` section added.
        - CONF: `search_indexation.include_mock_organizations` :bool
            Toggle to include mock organizations and corresponding endpoints in the generated search index output.
            Default: `false`.

        - CONF: `search_indexation.mock_organizations_path` :path
            Path to a JSON file containing mock organizations that can be merged into `organizations.json` when mock organizations are enabled.
            Default: `resources/search_index/mock-organizations.json`.
            Must point to an existing file when `search_indexation.include_mock_organizations=true`.

        - CONF: `search_indexation.mock_addressing_path` :path
            Path to a JSON file containing mock endpoint IDs and URLs used to generate encrypted endpoint entries in `endpoints.json` when mock organizations are enabled.
            Default: `resources/search_index/mock-endpoints.json`.
            Must point to an existing file when `search_indexation.include_mock_organizations=true`.

- CONF: `zorgab_scraper` section added.
    - CONF: `zorgab_scraper.zakl_path` :path|none
        Path to the ZAKL scrape source file; must persist across releases.
        Default: `None` (conditionally required when using `search-index:update` with `zakl_xml`).
    - CONF: `zorgab_scraper.agb_csv_path` :path|none
        Path to the AGB CSV scrape source file; must persist across releases.
        Default: `None` (conditionally required when using `search-index:update` with `agb_csv`).

- CONF: `jwe.encryption_public_key_path` :path added
        Path to the public key used for JWE encryption
        Required (no default).

- CONF: `normalization` section added (optional).
    - CONF: `normalization.normalization_output_folder` :str
        Output folder for normalization results.
        Default: `static/search/normalization`.

- CONF: `benchmark` section added (optional).
    - CONF: `benchmark.zorgab_write_output` :bool 
        Default: `false`.
    - CONF: `benchmark.zorgab_query_input_path` :str 
        Default: `resources/benchmark/zorgab/default_input.json`.
    - CONF: `benchmark.zorgab_output_dir` :str 
        Default: `storage/benchmarks/zorgab/`.

### Changed

- CONF: `signing` section renamed to `jwe`
    - CONF: `jwe.signing_private_key_path` :path
        Path to the private key used for JWT signing, generated by `./tools/generate-sign-key.sh`
        Required (no default).
        for more information on key generation see the script's help message.

- CONF: `app.mock_base_url` :str is now required.
    - Must be a fully qualified URL without trailing slash.
    - Required (no default).

### Removed
- CONF: `signing.sign_endpoints` :bool
    - Previous default: `true`.

The signing.sign_endpoints config setting actually has impact now.

## [0.1.0]

### Added

- CONF: [signing] section
  - A new section for the signing configuration
- CONF: signing.sign_endpoints
  - This is the feature flag for the signing functionality. If False imported ZAL endpoints will not be signed
- CONF: signing.private_key :str
  - The path to the private key that is used for signing
    - This key can be generated using the ./tools/generate-sign-key.sh cli script
- CONF: signing.param_name
  - The name for the signature query string parameter key

### Changed

- Change version from 0.0.4 to 0.1.0 to start using semver properly
